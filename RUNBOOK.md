# simple-auto — Session Runbook (internal)

Not published. Operational steps for running paid sessions.

## Before any session
- Confirm payment in PayPal for each attendee (match the email they booked with).
- Check each attendee filled the pre-session intake form (idea + uploaded assets in Drive).
- Have the lead Sheet open to cross-check leads ↔ bookings ↔ payments.

## Group session
Everyone works on **their own computer**; we guide the whole group live.

1. Kick off: confirm everyone has the coding agent (Codex / Antigravity) installed and a
   GitHub account.
2. Each attendee creates the repo and **adds us as a GitHub collaborator**
   (Repo → Settings → Collaborators → invite our GitHub user) so we can fix things if needed.
3. Walk the group step by step: scaffold the landing page → drop in their copy and assets →
   wire the CTA with **Web3Forms** → deploy → connect their domain.
4. Keep scope tight: one clean landing page + clear CTA. No logins, apps, or automations.
5. Ship check: every **paid** sign-up leaves with a live site. Friends watching are welcome
   but are not guaranteed a shipped site and yield to paid attendees for questions.
6. Share the session recording.

## Private session
One-on-one (friends may join). We **drive their computer over remote desktop**.

1. Start a remote-desktop session and take control of their screen.
2. Perform each step in place while narrating, so they can repeat it later.
3. Same deliverable as group, at their pace; add us as GitHub collaborator as above.
4. Share the recording.

## Ship checklist (per site)
- [ ] Site deployed and reachable on its URL.
- [ ] Custom domain connected (or noted as a follow-up; ~20 EUR/yr, theirs).
- [ ] CTA (Web3Forms) tested — a test submission arrives in their inbox.
- [ ] We're a GitHub collaborator on their repo.
- [ ] Recording shared.

## After
- Mark the lead row in the Sheet as shipped.
- Optional: offer the maintenance/improvement subscription.
